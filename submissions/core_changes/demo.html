Add rel="noopener noreferrer" to External Links

Issue #12192 — docs/index.html uses target="_blank" without rel="noopener noreferrer", creating a tabnabbing vulnerability.

Vulnerable Pattern (Before)

<a href="https://example.com" target="_blank">Link</a>

Secure Pattern (After)

<a href="https://example.com" target="_blank" rel="noopener noreferrer">Link</a>

Live Demo

These links demonstrate the secure pattern (noopener noreferrer added):

  • Discord Server
  • GitHub Repository

File Line Change
docs/index.html 112 Added rel="noopener noreferrer" to Discord link in header
